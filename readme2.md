# GWAS-PPI 解析パイプライン 仕様書兼設計図

本ドキュメント（`readme2.md`）は、GWASデータとPPI（タンパク質間相互作用）ネットワークを統合・解析し、創薬標的となる重要遺伝子を同定するためのパイプライン全体の設計・仕様をまとめた完全版の設計図です。

---

## 1. 構想と目的

疾患は「上流（ゲノム）」の変異が引き金となり、「下流（生体メカニズム）」のタンパク質群に異常なシグナル・相互作用（フロー）を引き起こすことで発症すると定義します。
本パイプラインの目的は、GWASデータに基づくゲノム情報（原因）と PPI ネットワーク情報（結果・メカニズム）を組み合わせ、**上流から下流へのシグナルの流れの中でハブ（中心）となる重要遺伝子を見つけ出すこと**（＝創薬標的の探索）です。

---

## 2. 解析パイプラインのフロー（Step-by-Step）

### Step 1: SNP および疾患関連遺伝子の特定 (`gwas_fetcher.py`)
特定の疾患・形質（Trait）に関連する SNP（一塩基多型）を調査し、それらが影響を及ぼす「SNP関連遺伝子（GWAS遺伝子）」を特定します。
- **データソース:** GWAS Catalog REST API
- **処理内容:**
  - 指定した Disease Trait (例: "Type 2 diabetes") に関連する有意な SNP を取得
  - P値 (`pvalueMantissa` × 10^`pvalueExponent`) を正確にパースし、重複を排除
  - API から `authorReportedGenes` 等を取得し、対象の遺伝子パネルをリスト化

### Step 2: PPI（タンパク質間相互作用）遺伝子群の取得 (`ppi_fetcher.py`)
Step 1 で特定した GWAS 遺伝子群をシードとして、それらと繋がりを持つ PPI ネットワークを構築します。
- **データソース:**
  - SIGNOR (シグナル伝達データベース / TSVを直接ダウンロードして手動パース)
  - Reactome (パスウェイ関係)
  - STRING (Score > 0.700 でフィルタリング)
  - BioGRID
- **拡張性:**
  - 分析対象のソースを選択可能（デフォルト: SIGNOR, Reactome, STRING）
  - ネットワークの深さ（階層）を 1〜3 階層から選択可能（デフォルト: 1階層）

### Step 3: フローベース・ネットワーク中心性解析 (`network_analysis.py`, `gene_scorer.py`)
GWAS 遺伝子を「水源（上流）」とし、PPI ネットワークを「川」と見立てて、各遺伝子への影響量（フロースコア）を計算します。

1. **初期流量（スコアリング）の定義 (`gene_scorer.py`):**
   - **GWAS P値スコア:** `-log10(p_value)`。ただし、他のスコアを圧倒しないよう **最大 10.0 にキャップ（正規化）**
   - **バリアント機能影響度スコア (VEP API):**
     - LoF (Loss of Function): +4.0
     - GoF (Gain of Function): +3.5
     - Missense: +2.0
     - Coding Variant: +1.5
     - Regulatory: +1.0
   - **PPI 信頼度スコア:** 複数のDB（ソース）で支持されているエッジほど高く評価
2. **ネットワーク上のフロー計算 (`network_analysis.py`):**
   - ランダムウォーク（PageRankアルゴリズムのカスタマイズ）を利用
   - 上記の初期流量（スコア）をノードの初期重みとし、PPI ネットワーク上を伝播させる
   - フローだけでなく、次数中心性等の標準的なネットワーク指標も算出し、統合スコアを決定して「Key PPI 遺伝子」を抽出

### Step 4: ローカル・エンリッチメント解析 (`enrichment.py`)
抽出された Key 遺伝子群が、どのような生物学的機能やパスウェイに関与しているかを解析します。
- **データソース:**
  - **Reactome:** GMTファイル
  - **Gene Ontology (GO):** MSigDB GMTファイル (`c5.go.v2024.1.Hs.symbols.gmt`)
  - **Human Phenotype Ontology (HPO):** `genes_to_phenotype.txt`
- **解析手法:** Fisherの正確確率検定 (Fisher's exact test)
- **多重検定補正:** Benjamini-Hochberg (BH) 法による FDR (False Discovery Rate) 算出

### Step 5: 高度な可視化とレポーティング (`visualization.py`)
解析結果を Jupyter Notebook 上でインタラクティブな図（Plotly）として出力し、結果を直感的に解釈できるようにします。

1. **ネットワーク図 (Spring Layout):**
   - 全遺伝子を力学モデル（反発力を強くした Spring Layout）で展開し、繋がりの中心を一目で把握可能にする。
   - **マーカー区別:** 
     - **上流 (GWAS遺伝子):** ◆ ダイヤモンド / 濃紺
     - **中心 (Key PPI遺伝子):** ★ スター / オレンジ赤
     - **下流 (その他PPI):** ◯ サークル / ライトブルー
   - **エッジ区別:** GWAS⇔PPI は太い青線、PPI同士は細いオレンジ線
2. **パスウェイ - 遺伝子スコアマトリックス:**
   - 抽出された重要パスウェイに対し、どの遺伝子が寄与しているかを示すヒートマップ
   - 単なる該当有無 (1/0) ではなく、**その遺伝子のフロースコア等に応じた色の濃淡** と数値を表示
3. **データソース別エンリッチメント結果:**
   - Reactome / GO / HPO それぞれを独立した **バブルチャート** および **棒グラフ (-log10 FDR)** として描画
4. **Sankey ダイアグラム:**
   - `SNP → GWAS遺伝子 → Key PPI遺伝子 → パスウェイ` への情報の流れを可視化（横幅900pxの長方形・省スペースで見やすいレイアウト）

## 3. 実行環境とモジュール構成

- **`config.py`**: 全APIのエンドポイント、DB設定、スコア重み付け等の定数管理
- **`id_mapper.py`**: UniProt API 等を利用した ID (Entrez/Symbol/UniProt) 相互変換
- **`notebooks/gwas_ppi_analysis.ipynb`**: ユーザーがパラメータを調整し、Step 1〜5 のモジュールを統合して実行するメインノートブック
- **結果出力先**: 全ての可視化 HTML、および中間データ (CSV) は `output/` フォルダに出力され、いつでもレポーティングに利用可能
